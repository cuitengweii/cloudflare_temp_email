# 线程决策归档（DECISIONS）

## 2026-04-21 | 员工邮箱产品路径稳定决策
1. 员工账号不再走公开注册。
   - 正式站前台不再暴露注册入口。
   - 线上配置默认关闭 `enable`，即关闭公开注册接口。

2. 员工账号统一由管理员创建。
   - 管理后台创建用户时自动生成随机密码。
   - 管理员直接将“员工邮箱 + 初始密码”交付给员工。

3. 忘记密码继续保留。
   - 即使关闭公开注册，只要 `enableMailVerify = true`，员工仍可通过邮箱验证码重置密码。
   - 公开注册和忘记密码是两条独立能力，不能继续绑定为同一开关语义。

4. 员工邮箱域名限制继续保留公司域白名单。
   - 当前允许域名仅保留 `gasgx.com`。
   - 验证邮件发送地址也必须属于系统域名。

5. 新建邮箱默认可发信。
   - 新邮箱创建后自动补齐默认发信权限。
   - 管理员仍可在后台继续手动禁用或调整余额。

## 2026-04-14 | 部署与暗色模式相关稳定决策
1. 前端发布必须从 `pages` 目录执行 `wrangler pages deploy`。
2. 暗色模式 active 态统一采用透明背景 + 底部高亮线，不再使用大面积亮色背景块。
3. 执行 `wrangler` 前优先清理代理变量，并在需要时强制 `ipv4first` 规避 Cloudflare 认证超时。

# 线程决策归档（DECISIONS）

## 2026-04-21 | 员工邮箱账号恢复路径最终收口
1. 员工账号不再提供自助忘记密码。
   - 员工登录页不再展示“忘记密码”入口。
   - 员工不能再通过验证码自行重置密码。

2. 员工账号统一由管理员创建并下发初始密码。
   - 管理后台创建用户时默认生成随机密码。
   - 管理员直接将“员工邮箱 + 临时密码”交付给员工。

3. 员工登录后的自主能力仅保留“已登录后修改密码”。
   - 这属于账户内主动变更，不属于遗忘密码恢复流程。
   - 因此继续保留，不与“忘记密码”收口决策冲突。

4. 遗忘密码时唯一受支持路径为管理员后台重置。
   - 管理后台“重置密码”默认生成新的随机临时密码。
   - 重置成功后立即弹出新的登录凭证，便于管理员复制并重新下发。

5. `user_api/verify_code` / `user_api/register` 不再承担员工密码找回职责。
   - 即使底层仍保留邮件验证码基础设施，当前产品流也不再暴露该能力。
   - 接口层直接返回禁用提示，避免后续被前端或第三方误用。

6. 员工邮箱域名限制继续保留公司域白名单。
   - 当前允许域名仍为 `gasgx.com`。
   - 该限制用于员工账号边界，不因密码恢复流程调整而改变。

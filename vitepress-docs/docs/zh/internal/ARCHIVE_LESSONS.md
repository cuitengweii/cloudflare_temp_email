# 线程经验归档（LESSONS）

## 2026-04-21 | 本轮踩坑与规避
1. “公开注册关闭”和“员工忘记密码”不能继续混用同一套产品语义。
   - 如果员工账号本来就是管理员预创建并下发初始密码，那么“忘记密码”继续走公开验证码链路会让边界混乱。
   - 规避：
     - 公开前台只保留登录入口。
     - 遗忘密码统一回到管理员后台处理。

2. 管理员建号与管理员重置密码必须统一为同一套交付方式。
   - 建号时已经是“随机密码 + 复制下发”，若重置密码仍要求管理员手输，会造成体验和口径不一致。
   - 规避：
     - 重置密码同样默认生成随机临时密码。
     - 重置成功后直接展示“邮箱 + 新密码”供复制。

3. 公开接口即使前端不再使用，也应显式禁用而不是默默保留。
   - 仅删前端入口会留下后端误用空间，后续容易被旧脚本或旧页面继续调用。
   - 规避：
     - `user_api/verify_code` 和 `user_api/register` 直接返回禁用提示。
     - 测试同步覆盖“员工自助找回被禁用”的行为。

4. 内部归档文档一旦出现编码损坏，应优先重写为 UTF-8 可维护版本。
   - 否则后续线程很难可靠追加最新事实。
   - 规避：
     - 本次直接重写 STATE / DECISIONS / ARCHITECTURE / LESSONS，保留最新有效事实。

# 线程经验归档（LESSONS）

## 2026-04-21 | 本轮踩坑与规避
1. “员工邮箱验证码注册”非常容易被误解。
   - 用户直觉会认为：填写一个公司邮箱字符串，就应该能收到验证码。
   - 真实前提却是：该邮箱必须先存在、能收件、并且员工能访问它。
   - 规避：
     - 产品路径改为“管理员预先创建账号并下发初始密码”。
     - 不再让员工面对“先填邮箱、再等验证码”的自助注册路径。

2. 关闭公开注册后，忘记密码不能一起误伤。
   - 之前前端把“能否注册”和“能否重置密码”绑定成同一条件，语义容易错。
   - 规避：
     - 忘记密码只依赖 `enableMailVerify`。
     - 公开注册由 `enable` 单独控制。

3. Pages 发布容易被本机代理或 IPv6 直连影响。
   - 症状：
     - `403 Forbidden`
     - bot challenge
     - `UND_ERR_CONNECT_TIMEOUT`
   - 规避：
     - 部署前清空 `HTTP_PROXY / HTTPS_PROXY / ALL_PROXY`
     - 必要时设置 `NODE_OPTIONS=--dns-result-order=ipv4first`

4. 旧归档文档存在编码损坏。
   - 症状：VitePress 内部归档文档出现乱码，无法可靠追加。
   - 规避：
     - 本次直接重写为 UTF-8 可读版本，只保留仍有效的状态、决策、架构和经验。
